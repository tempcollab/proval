## Current best

Notation: `N = k^2 + 1`, `S` = set of `(k+1)`-tuples `1 <= x_1 < ... < x_{k+1} <= N`.
Read a permutation `sigma` of `[N]` as the sequence `a_p = sigma(p)`; a `sigma`-nice
tuple is exactly a monotone (increasing or decreasing) subsequence of `a` of length
`k+1` indexed by an element of `S`. Write `L(X), R(X)` for the two `min`-of-floors.

The following are **established rigorously** (and machine-verified for `k=2,3,4`):

**Lemma 0 (staircase reformulation of L and R).**
For an integer threshold `t >= 1` and `i >= 1`, `floor(x_i / i) >= t  <=>  x_i >= t*i`
(divide the integer inequality `x_i >= t*i` by `i`; conversely `floor(x_i/i) >= t` gives
`x_i >= floor(x_i/i)*i >= t*i`). Taking the `min` over `i` and the largest clearing `t`:
- `L(X) = max{ t : x_i >= t*i for all i = 1..k }`.
- Likewise, since `floor((N+1 - x_i)/(k+2-i)) >= t <=> N+1-x_i >= t*(k+2-i)`,
  `R(X) = max{ t : N+1 - x_i >= t*(k+2-i) for all i = 2..k+1 }`.

(Note `k^2 + 2 - x_i = N + 1 - x_i`; the numerator is `N+1`, not `N` or `N+2`.)
So `L(X) = a` means `X` lies on or above the bottom `a`-staircase `(x_i >= a*i)`, and
`R(X) = b` means `X` lies on or below the top `b`-staircase. **Verified: these formulas
reproduce the competition floor-values with zero mismatches for all tuples, k=2,3,4.**

**Lemma A (range of a, b).** For every `X in S`, `1 <= L(X) <= k` and `1 <= R(X) <= k`.
*Proof.* The `x_i` are strictly increasing integers `>= 1`, so `x_i >= i`, hence
`floor(x_i/i) >= 1` and `L(X) >= 1`; the `rho`-mirror gives `R(X) >= 1`. If `L(X) >= k+1`
then `x_k >= (k+1)k = k^2 + k > k^2 + 1 = N` (as `k > 1`), impossible; so `L(X) <= k`, and
by mirror `R(X) <= k`. **Verified: min(L,R)=1, max(L,R)=k for k=2,3,4.** âˆŽ

**Lemma R (reflection).** The map `rho(x) = k^2 + 2 - x = N + 1 - x` is an
order-reversing involution of `[N]`. For `X in S` put `rho(X) = (N+1-x_{k+1}, ...,
N+1-x_1) in S`. Then `L(rho X) = R(X)` and `R(rho X) = L(X)`. If `tau = rho . sigma . rho`
(equivalently the sequence `a` read with positions and values both reflected), then `X` is
`sigma`-nice iff `rho(X)` is `tau`-nice, with the monotonicity direction reversed, and `X`
is the unique `sigma`-nice tuple iff `rho(X)` is the unique `tau`-nice tuple. Consequently:
- (Sufficiency) it suffices to build an isolating `sigma` realizing `X` as the unique
  *increasing* nice tuple; the decreasing case follows by applying `rho`.
- (Necessity) given an isolating `sigma`, after possibly replacing `sigma` by `tau` we may
  assume `X` is realized *increasingly*. **Verified: L(rho X)=R(X), R(rho X)=L(X) exactly,
  k=2,3,4.**

**Patience-sort coordinates (knowledge_base.md, "Monotone Subsequences").**
For the sequence `a`, attach `(I_p, D_p) =` (longest increasing run ending at `p`, longest
decreasing run ending at `p`). Then (C1) `p -> (I_p, D_p)` is injective, (C2)
`max_p I_p = LIS(a)`, `max_p D_p = LDS(a)`, and (ES) since `N = k^2+1 > k^2`, every `sigma`
has `LIS >= k+1` or `LDS >= k+1` â€” so a `sigma`-nice tuple always exists; the problem is
about *uniqueness*.

**Grid skeleton for the construction (verified, not yet a closed-form proof).**
Writing `a = L(X)`, `b = R(X)`, the increasing-realization construction must place the
`k+1` points of `X` on a single increasing "spine" at coordinate row `D = a` (constant `D`,
`I = 1..k+1`), with the `k^2 - k` complement points confined to the `k x k` box
`{1..k} x {1..k}` of `(I,D)`-cells: `a-1` complement rows above the spine and `k - a`
rows below it, each row holding `k` cells. The constraint that exactly `k - a` rows fit
below (each needing `D <= k`) is `k - a <= b - 1`, i.e. **`a + b >= k+1`** â€” the hypothesis,
exactly. Then `LIS = k+1` (the spine) and `LDS <= k`, so the only monotone `(k+1)`-runs
are increasing, and the a/b band split forces the unique increasing transversal to be `X`.

**Status of the construction.** A deterministic row-family search realizes this skeleton
and isolates **all** good tuples for `k=2` (4/4) and `k=3` (49/49), including interior

all `(a,b)` with `a+b >= k+1` together with a proof that `X` is the *unique* increasing

the *fixed* position set `x_1 < ... < x_{k+1}`: the realized `(I_p, D_p)` are outputs of the
chosen sequence and cannot be set freely.
