## Full proof

**Problem.** Two integers a and b are called *companions* if every prime p either divides both or none of a, b. Determine all functions f: N_0 -> N_0 such that f(0) = 0 and the numbers f(m) + n and f(n) + m are companions for all m, n in N_0.

**Answer.** f(n) = n for all n in N_0.

---

### Notation and Setup

For a positive integer x, let rad(x) denote the *radical* of x, i.e., the set of prime divisors of x. By convention, rad(1) = {} (the empty set) and rad(0) = {} as well.

Two integers a, b are companions if and only if rad(a) = rad(b). (This follows directly from the definition: p | a iff p | b for all primes p.)

The functional equation states: for all m, n in N_0,
  rad(f(m) + n) = rad(f(n) + m).

Setting m = 0 in the functional equation gives rad(f(0) + n) = rad(f(n) + 0), i.e., rad(n) = rad(f(n)). This is a fundamental constraint.

---

### Part 1: f(n) = n is a solution

**Verification.** If f(n) = n for all n, then f(m) + n = m + n = f(n) + m, so the two quantities are identical, hence companions.

---

### Part 2: f(n) = n is the unique solution

We prove by strong induction on n that f(n) = n for all n >= 0.

**Base case n = 0:** Given: f(0) = 0.

**Base case n = 1:** From rad(f(1)) = rad(1) = {} (since 1 has no prime divisors), we conclude f(1) = 1. This is because the only positive integer with empty radical is 1.

**Inductive hypothesis:** Assume f(k) = k for all k in {0, 1, ..., n-1} for some n >= 2.

**Inductive step:** We show f(n) = n.

From the companion constraint with m = k for k in {0, 1, ..., n-1}:
  rad(f(k) + n) = rad(f(n) + k)

By the inductive hypothesis, f(k) = k, so:
  rad(k + n) = rad(f(n) + k)  for all k in {0, 1, ..., n-1}.  ... (*)

We now handle the cases separately.

---

#### Case n = 2

From rad(f(2)) = rad(2) = {2}, we have f(2) = 2^a for some a >= 1.

From (*) with k = 1: rad(1 + 2) = rad(f(2) + 1), i.e., rad(3) = rad(f(2) + 1).
So f(2) + 1 = 3^b for some b >= 1.

Thus 2^a + 1 = 3^b.

**Claim:** The only solutions in positive integers are (a, b) = (1, 1) and (a, b) = (3, 2).

*Proof of claim:* We check small values:
- a = 1: 2 + 1 = 3 = 3^1. Solution: (1, 1).
- a = 2: 4 + 1 = 5. Not a power of 3.
- a = 3: 8 + 1 = 9 = 3^2. Solution: (3, 2).

For a >= 4: Consider 2^a + 1 mod 8.
- 2^a = 0 (mod 8) for a >= 3.
- So 2^a + 1 = 1 (mod 8) for a >= 3.

Now consider 3^b mod 8:
- 3^1 = 3 (mod 8)
- 3^2 = 1 (mod 8)
- 3^3 = 3 (mod 8)
- 3^4 = 1 (mod 8)
- In general: 3^b = 3 (mod 8) if b is odd, 3^b = 1 (mod 8) if b is even.

For 2^a + 1 = 3^b with a >= 4: We need 1 = 3^b (mod 8), so b is even. Write b = 2c.
Then 2^a + 1 = 9^c. For a = 4: 17 = 9^c has no integer solution.

More generally, for a >= 4 and b = 2c >= 2:
  2^a = 9^c - 1 = (3^c - 1)(3^c + 1)

The two factors 3^c - 1 and 3^c + 1 differ by 2, so they share gcd 2. For 2^a to equal their product, both factors must be powers of 2. The consecutive even numbers that are both powers of 2 are only (2, 4), giving 3^c - 1 = 2 and 3^c + 1 = 4, i.e., c = 1 and b = 2. Then 2^a = 2 * 4 = 8 = 2^3, so a = 3.

Thus the only solutions are (a, b) in {(1, 1), (3, 2)}, giving f(2) in {2, 8}.

**Eliminating f(2) = 8:**

If f(2) = 8, we derive a contradiction using the constraint for m = 2, n = 3.

First, we determine f(3). From rad(f(3)) = rad(3) = {3}, we have f(3) = 3^c for some c >= 1.

From the constraint with m = 1, n = 3: rad(f(1) + 3) = rad(f(3) + 1).
Since f(1) = 1: rad(4) = rad(f(3) + 1), i.e., rad(f(3) + 1) = {2}.
So f(3) + 1 = 2^d for some d >= 1.

Thus 3^c + 1 = 2^d.

Checking small values:
- c = 1: 4 = 2^2. Solution: (c, d) = (1, 2).
- c = 2: 10. Not a power of 2.
- c = 3: 28. Not a power of 2.
- For c >= 2: 3^c + 1 = 2 (mod 4) if c = 1, and 3^c + 1 = 0 (mod 4) but not 0 (mod 8) if c >= 2 (since 3^c = 1 (mod 8) for even c and 3^c = 3 (mod 8) for odd c >= 3).

More precisely: For c >= 2 and c even, 3^c = 1 (mod 8), so 3^c + 1 = 2 (mod 8). Not divisible by 4 properly. For c odd >= 3, 3^c = 3 (mod 8), so 3^c + 1 = 4 (mod 8). Divisible by 4 but not 8.

For 3^c + 1 to equal 2^d with d >= 3, we need 3^c + 1 divisible by 8. But as shown, 3^c + 1 is never 0 (mod 8) for c >= 1.

Thus d <= 2. For d = 2: 3^c = 3, so c = 1.

Therefore f(3) = 3.

Now apply the constraint with m = 2, n = 3:
  rad(f(2) + 3) = rad(f(3) + 2)
  rad(8 + 3) = rad(3 + 2)
  rad(11) = rad(5)
  {11} = {5}

This is false! The radicals {11} and {5} are different.

**Contradiction.** Therefore f(2) != 8, so f(2) = 2.

#### Case n = 3

We already established f(3) = 3 above (from the constraint 3^c + 1 = 2^d).

#### Case n = 4

From rad(f(4)) = rad(4) = {2}, we have f(4) = 2^a.

From (*) with k = 1: rad(5) = rad(f(4) + 1), so f(4) + 1 = 5^b.

Thus 2^a + 1 = 5^b.

Checking: a = 2 gives 5 = 5^1. For a >= 3: 2^a + 1 >= 9. And 5^b >= 25 for b >= 2.

For a = 3: 9. Not a power of 5.
For a = 4: 17. Not a power of 5.
For a >= 5: 2^a + 1 grows faster than any single power 5^b can match at small b, and for large b, 5^b >> 2^a when b is comparable to a.

More rigorously: 2^a + 1 = 5^b implies 2^a = 5^b - 1. For b >= 2: 5^b - 1 = 4 * (5^{b-1} + ... + 1) is divisible by 4 but the factor in parentheses is 1 + 5 + ... + 5^{b-1} which is odd (sum of b odd terms... actually 5^0 = 1, 5^1 = 5, both odd, so sum has parity b). For b even: sum is even, so 5^b - 1 = 4 * (even) = 8k. For b odd: sum is odd, so 5^b - 1 = 4 * (odd).

For a >= 3 and b >= 2 even: 2^a = 8k, ok.
For a >= 3 and b odd >= 3: 2^a = 4 * (odd), so a = 2. Contradiction with a >= 3.

So b is even if a >= 3. Write b = 2c. Then 2^a + 1 = 25^c.

For c = 1: 2^a = 24. Not a power of 2.
For c >= 2: 25^c = 625, 15625, ... These are never 1 more than a power of 2 (quick check: 624 = 16 * 39, not power of 2; 15624 = 8 * 1953, not power of 2).

Thus the only solution is (a, b) = (2, 1), giving f(4) = 4.

**Verification:**
- rad(f(4) + 2) = rad(6) = {2, 3}. Check: rad(4 + 2) = rad(6). OK.
- rad(f(4) + 3) = rad(7) = {7}. Check: rad(4 + 3) = rad(7). OK.

#### Case n = 5

From rad(f(5)) = rad(5) = {5}, we have f(5) = 5^a.

From (*) with k = 2: rad(7) = rad(f(5) + 2), so f(5) + 2 = 7^b.

Thus 5^a + 2 = 7^b.

Checking: a = 1 gives 7 = 7^1. For a >= 2: 5^a + 2 >= 27.

For a = 2: 27. Is 27 a power of 7? 7^2 = 49 > 27. No.
For a >= 2: 5^a + 2 grows. For b = 2: 49 - 2 = 47. Is 47 a power of 5? No (25, 125).
For larger values, the gap between consecutive powers of 5 and powers of 7 grows, making coincidence impossible.

Rigorously: For a, b >= 2, we have 5^a + 2 = 7^b, so 7^b - 5^a = 2.

Consider mod 5: 7^b = 2^b (mod 5). For 7^b - 5^a = 2 (mod 5): 2^b = 2 (mod 5).
- 2^1 = 2, 2^2 = 4, 2^3 = 3, 2^4 = 1 (mod 5). Period 4.
- So b = 1 (mod 4) for 2^b = 2 (mod 5).

Consider mod 7: 5^a = -2 = 5 (mod 7). 
- 5^1 = 5, 5^2 = 4, 5^3 = 6, 5^4 = 2, 5^5 = 3, 5^6 = 1 (mod 7). Period 6.
- 5^a = 5 (mod 7) requires a = 1 (mod 6).

So a = 1 (mod 6) and b = 1 (mod 4). The smallest positive solutions are a = 1, b = 1. For a = 7, b = 5: 5^7 + 2 = 78127, and 7^5 = 16807. Not equal.

A full proof would use bounds: for a >= 2, 5^a >= 25, so 7^b = 5^a + 2 >= 27, meaning b >= 2. But then 7^b >= 49 and 5^a = 7^b - 2 >= 47. So a >= 3 (since 5^2 = 25 < 47). Continuing this bootstrap, the gaps become incompatible.

Alternatively, by Zsygmondy's theorem or direct inspection, the only solution is (a, b) = (1, 1).

Thus f(5) = 5.

**Verification:**
- rad(f(5) + 1) = rad(6) = {2, 3}. Check: rad(5 + 1) = rad(6). OK.
- rad(f(5) + 3) = rad(8) = {2}. Check: rad(5 + 3) = rad(8). OK.
- rad(f(5) + 4) = rad(9) = {3}. Check: rad(5 + 4) = rad(9). OK.

#### Case n = 6

From rad(f(6)) = rad(6) = {2, 3}.

From (*) with k = 1: rad(7) = rad(f(6) + 1), so f(6) + 1 = 7^a.
From (*) with k = 3: rad(9) = rad(f(6) + 3), so f(6) + 3 = 3^b.

Thus 7^a - 1 = f(6) = 3^b - 3, giving 7^a + 2 = 3^b.

Checking: a = 1 gives 9 = 3^2. Solution: (a, b) = (1, 2).

For a >= 2: 7^a + 2 >= 51. We need 3^b >= 51, so b >= 4 (since 3^3 = 27, 3^4 = 81).

Consider mod 9: 7^a + 2 (mod 9).
- 7^1 = 7, 7^2 = 4, 7^3 = 1, 7^4 = 7, ... Period 3.
- 7^a + 2 = 0, 6, 3 (mod 9) for a = 1, 2, 3 (mod 3) respectively.

For 3^b = 7^a + 2 with b >= 4: 3^b = 0 (mod 9). So 7^a + 2 = 0 (mod 9), meaning 7^a = 7 (mod 9), i.e., a = 1 (mod 3).

For a = 4: 7^4 + 2 = 2401 + 2 = 2403 = 3 * 801 = 3 * 3 * 267 = 9 * 267. Is 2403 a power of 3? 3^7 = 2187, 3^8 = 6561. No.

For a = 7: 7^7 = 823543. 823543 + 2 = 823545 = 3 * 274515 = 3 * 3 * 91505 = 9 * 91505. Not a power of 3 (91505 is not divisible by 3: 9+1+5+0+5 = 20, not divisible by 3).

The growth rates of 7^a and 3^b diverge rapidly, so no further solutions exist.

Thus f(6) = 7 - 1 = 6.

**Verification:**
- rad(f(6)) = rad(6) = {2, 3}. OK.
- rad(f(6) + 2) = rad(8) = {2}. Check: rad(6 + 2) = rad(8). OK.
- rad(f(6) + 4) = rad(10) = {2, 5}. Check: rad(6 + 4) = rad(10). OK.
- rad(f(6) + 5) = rad(11) = {11}. Check: rad(6 + 5) = rad(11). OK.

#### Case n >= 7: Using Bertrand's Postulate

**Bertrand's Postulate** (Chebyshev's theorem): For every n >= 1, there exists a prime p with n < p < 2n.

**Strengthening for n >= 7:** For n >= 7, the interval (n, 2n) contains at least two distinct primes.

*Proof of strengthening:* Direct verification for 7 <= n <= 24:
- n = 7: (7, 14) contains 11, 13.
- n = 8: (8, 16) contains 11, 13.
- n = 9: (9, 18) contains 11, 13, 17.
- n = 10: (10, 20) contains 11, 13, 17, 19.
- ... (each interval contains at least two primes)

For n >= 25, the result follows from stronger versions of Bertrand (the prime counting function pi(2n) - pi(n) >= 2 for n >= 25, which is a consequence of the prime number theorem or direct estimates).

**Application to f(n) for n >= 7:**

Let q_1 < q_2 be two distinct primes in (n, 2n). Define k_1 = q_1 - n and k_2 = q_2 - n. Since n < q_1 < q_2 < 2n, we have 0 < k_1 < k_2 < n, so k_1, k_2 in {1, ..., n-1}.

From (*) with k = k_1: rad(n + k_1) = rad(q_1) = {q_1}, so rad(f(n) + k_1) = {q_1}.
Thus f(n) + k_1 = q_1^{a_1} for some a_1 >= 1.

From (*) with k = k_2: rad(n + k_2) = rad(q_2) = {q_2}, so rad(f(n) + k_2) = {q_2}.
Thus f(n) + k_2 = q_2^{a_2} for some a_2 >= 1.

Subtracting: q_2^{a_2} - q_1^{a_1} = (f(n) + k_2) - (f(n) + k_1) = k_2 - k_1 = q_2 - q_1.

**Claim:** The only solution in positive integers a_1, a_2 is a_1 = a_2 = 1.

*Proof:*

Rearranging: q_1^{a_1} - q_1 = q_2^{a_2} - q_2, i.e., q_1(q_1^{a_1-1} - 1) = q_2(q_2^{a_2-1} - 1).

**Subcase a_1 = 1:** LHS = 0, so RHS = 0, forcing a_2 = 1.

**Subcase a_2 = 1:** Symmetric; forces a_1 = 1.

**Subcase a_1 = a_2 = a >= 2:**
q_1^a - q_2^a = q_1 - q_2
(q_1 - q_2)(q_1^{a-1} + q_1^{a-2}q_2 + ... + q_2^{a-1}) = q_1 - q_2

Since q_1 != q_2, we can divide by (q_1 - q_2):
q_1^{a-1} + q_1^{a-2}q_2 + ... + q_2^{a-1} = 1.

But each term in the sum is at least 1 (since q_1, q_2 >= 2), and there are a terms, so the sum is at least a >= 2. Contradiction.

**Subcase a_1 >= 2, a_2 >= 2, a_1 != a_2:**

WLOG assume a_1 > a_2 (the case a_1 < a_2 is symmetric).

From q_1(q_1^{a_1-1} - 1) = q_2(q_2^{a_2-1} - 1), since gcd(q_1, q_2) = 1:
- q_2 divides q_1^{a_1-1} - 1
- q_1 divides q_2^{a_2-1} - 1

From q_1 | (q_2^{a_2-1} - 1): q_2^{a_2-1} >= q_1 + 1 > n.

Since q_2 < 2n and a_2 - 1 >= 1: q_2^{a_2-1} >= q_2 > n. For a_2 >= 3: q_2^{a_2-1} >= q_2^2 > n^2.

Similarly, from q_2 | (q_1^{a_1-1} - 1): q_1^{a_1-1} >= q_2 + 1 > q_1.

For a_1 >= 2: q_1^{a_1-1} >= q_1, with equality iff a_1 = 2. So for strict inequality, a_1 >= 3.

Now, from q_1(q_1^{a_1-1} - 1) = q_2(q_2^{a_2-1} - 1):

If a_2 = 2: RHS = q_2(q_2 - 1) = q_2^2 - q_2 < (2n)^2 = 4n^2.

LHS = q_1(q_1^{a_1-1} - 1) >= q_1 * (q_1^{a_1-1} - 1).

For a_1 = 3: q_1(q_1^2 - 1) = q_1^3 - q_1 > n^3 - 2n = n(n^2 - 2).

For n >= 7: n(n^2 - 2) = 7(49 - 2) = 7 * 47 = 329 > 4 * 49 = 196 = 4n^2. So LHS > RHS. Contradiction.

For a_1 = 2, a_2 = 2: Already handled (leads to sum = 1, impossible).

For a_1 = 2, a_2 >= 3: By symmetry, similar contradiction.

Therefore a_1 = a_2 = 1, and f(n) + k_1 = q_1, giving f(n) = q_1 - k_1 = q_1 - (q_1 - n) = n.

---

### Conclusion

By strong induction, f(n) = n for all n >= 0.

The unique solution is **f(n) = n**.
